# Deno Demo Application

I'll create a Deno application that demonstrates various features of the Deno runtime. This will be a single-page web application that showcases Deno's capabilities including:

1. HTTP server functionality
2. File system operations
3. API fetching and JSON handling
4. TypeScript integration
5. Third-party module usage
6. Environment variables
7. Web standards support

The application will feature a simple UI that allows users to interact with these different features through a web browser, with real-time results display.

## File Tree

```
/deno-demo/
├── main.ts                 # Entry point for the application, sets up HTTP server
│                           # Import: import { startServer } from "./server.ts";
│
├── server.ts               # HTTP server implementation
│                           # Import: import { router } from "./routes.ts";
│
├── routes.ts               # API route definitions and handlers
│                           # Import: import { fileOps } from "./features/fileOps.ts";
│                           # Import: import { apiExample } from "./features/apiExample.ts";
│
├── static/                 # Static assets directory
│   ├── styles.css          # CSS styling for the demo app
│   ├── app.js              # Client-side JavaScript
│   └── index.html          # Main HTML page structure
│
└── features/               # Feature demonstrations
    ├── fileOps.ts          # File system operations demo
    │                       # Import: import * as path from "https://deno.land/std/path/mod.ts";
    │
    ├── apiExample.ts       # External API interaction demo
    │                       # Import: import { delay } from "https://deno.land/std/async/delay.ts";
    │
    └── envVars.ts          # Environment variables demo
                            # No imports needed
```

This structure keeps things simple with minimal nesting while providing a clear organization for the different components of the demo application.
